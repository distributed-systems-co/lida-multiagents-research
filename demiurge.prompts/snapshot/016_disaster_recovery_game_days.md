Title: Disaster Recovery Game Days
Domain: snapshot
Mode: audit

Prompt:
- Run periodic DR game days using snapshots and restore drills.
- Define scenarios (loss, corruption, region) and SLOs.
- Measure RTO/RPO; capture gaps; file fixes.
- Involve cross‑functional teams; protect real users.
- Publish learnings; update runbooks.
- Verify transparency and evidence.
- Schedule cadence and increasing difficulty.
- Archive results.

Outputs:
- GAME DAY scenarios and SLOs.
- RESULTS and remediation list.
- RUNBOOK updates and schedule.

Assumptions:
- Safe staging environments exist; leadership support.
- Monitoring/alerts reliable; rollback safe.
- Evidence capture is tamper‑evident.

Metrics to watch:
- RTO/RPO, pass rate, regression count.
- Participation and readiness scores.
- Time‑to‑fix and re‑test outcomes.
